# 안심e군

군 성군기 위반 ZERO 프로젝트 — 성군기 위반 신고 · 상담 · 건의 접수 앱

## 주요 기능
- **시작화면**: 부대 선택 (571대대 / 572대대 / 정비근무대 / 장비중대)
- **성군기 위반이란?**: 관련 법령, 실제 판례 3건, 가상 사례 재연 영상(재생 중 BGM 자동 음소거)
- **신고 / 상담 / 건의 접수**: 제목·내용 작성, 파일 첨부, 음성 녹음 첨부(최대 3분)
  - 제출 시 6자리 접수번호 발급 → [설정 > 민원 처리상태 확인]에서 익명 조회
- **관리자 (부대별 계정 분리)**:
  - 자기 부대 접수 목록 열람 (미확인 건 🔴 NEW 표시, 입장 시 알림)
  - 열람 시 자동 "확인 완료" 처리, 답변 작성(제출자가 접수번호로 조회 가능)
  - 이미지·음성 첨부 앱 내 재생, 처리 완료(삭제)
  - 비밀번호 변경, 상담관 전화번호 변경 (모든 기기에 적용)
  - 5분 무조작 또는 화면 이탈 시 자동 로그아웃
- **관리자 알림**: 관리자 로그인한 기기는 15분 주기로 새 민원을 확인해 알림 표시
- **전송 대기함**: 네트워크가 안 될 때 제출 내용을 기기에 저장, 연결되면 자동 전송
- **보관함**: "처리 완료(보관)" → 보관함으로 이동 (복원/완전삭제 가능, 사후 확인용)
- **임시저장**: 신고·상담 작성 중 앱이 꺼져도 제목·내용 자동 보존
- **긴급 도움 연결**: 국방헬프콜 1303, 여성긴급전화 1366, 경찰 112, 부대 상담관
- **BGM**: bgm1→2→3 순환(5초 간격), 화면 전환에도 유지, 설정에서 볼륨 조절,
  신고·상담 화면 자동 음소거 옵션

## 서버 (Firebase)
- 접수 내역·관리자 비밀번호·상담관 번호는 Firebase Firestore(프로젝트: zeroproject-93846)에
  저장되어 **기기 간 공유**됩니다. (병사 폰에서 제출 → 간부 폰에서 열람)
- 다른 Firebase 프로젝트로 바꾸려면 `FirebaseConfig.kt`의 PROJECT_ID / API_KEY를 교체하고,
  Firestore 규칙을 `allow read, write: if true;`로 게시하세요.
- ⚠ 보안: 위 규칙은 부대 내부용 소규모 운영을 위한 절충안입니다. 민감한 실명 정보를
  본격적으로 다루려면 Firebase Authentication 등 보안 강화가 필요합니다.

## APK 빌드 방법
1. **GitHub Actions (권장)**: main 브랜치에 push하면 자동 빌드
   → 저장소 Actions 탭 → 완료된 실행 → Artifacts에서 `ansim-e-gun-debug-apk` 다운로드
2. **Android Studio**: 이 폴더를 열고 Run ▶

## 배포 전 확인사항
- 부대별 관리자 기본 비밀번호(`Units.kt`)를 변경하거나, 앱의 [비밀번호 변경] 기능으로 교체
- 관리자 화면에서 부대 상담관 전화번호 등록
- "성군기 위반이란?" 법령 내용을 법제처(law.go.kr) 최신 조문과 대조

## 주요 파일
- `StartActivity` 시작화면(부대선택) / `MainActivity` 메인 / `InfoActivity` 법령·사례·영상
- `SubmitActivity` 신고·상담·건의 작성 / `SettingsActivity` 설정·상태조회
- `AdminLoginActivity` · `AdminListActivity` 관리자
- `RemoteStore` 서버 통신 / `FirebaseConfig` 서버 연결정보 / `Units` 부대 목록·기본 비밀번호
- `BgmPlayer` · `App` 배경음악 / `Emergency` 긴급연결
